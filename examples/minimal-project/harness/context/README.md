# Context

## 用途

这个文件夹是**项目上下文事实源**：系统架构、开发环境、外部集成、调研材料等 agent 改动前需要理解的背景。它按主题分子目录组织，每个子目录自带更细的 guide。

## 怎么用

- 按子目录取用，别整包读：
  - `architecture/` — 系统结构事实源（当前仓库负责什么、属于哪个更大系统、关键服务/流程/架构决策）。详见 `architecture/README.md`。
  - `development/` — 本地开发/构建/运行环境说明。
  - `integrations/` — 与外部系统/服务的集成契约与说明。
  - `research/` — 调研包与专题材料（含日期化的调研目录）。
- 本 README 只做导航与分类，**不复述**各子目录内容；具体事实以各子目录内的文档为准（D3 指针不复述）。

## 放什么 / 不放什么

- ✅ 放：长期有效、跨任务复用的背景事实，按上述主题归类。
- ❌ 不放：承重架构选择的"为什么"（→ `../decisions/` + `../adr/`）、任务级临时笔记（→ task package）、规则规范（→ `../standards/`）。
