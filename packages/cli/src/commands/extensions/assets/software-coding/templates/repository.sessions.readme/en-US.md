# Sessions

## 用途

This folder is the **session provenance archive**: each `*.md` is an export snapshot of one agent/human session (`provenance-session/v1`), recording runtime, source, timestamp, and session body. It makes "which session produced this decision/change" traceable.

## 怎么用 (How to use)

- Files here are generated by the harness session-export mechanism — **do not hand-write session files**.
- Decisions/tasks point back to a session via the frontmatter `provenance` (`runtime` + `sessionId` + `boundAt`), forming the provenance chain.
- To review "how this choice was reasoned at the time", follow an entity's `provenance.sessionId` to the matching `<sessionId>.md`.

## 放什么 / 不放什么 (What goes here / what does not)

- ✅ Put: session export snapshots (generated by command).
- ❌ Do not put: hand-written notes, decision bodies, task progress. This is a provenance ledger only; it carries no semantic decisions.
- ❌ Do not put: knowledge that needs long-term retrieval (that belongs to fact / decision / context).

## 相关命令 (Related commands)

Sessions are produced by the export flow (e.g. `ha session export ...` / automatic runtime export); the entity side binds them via the `provenance` field, not by hand-maintaining this folder.
