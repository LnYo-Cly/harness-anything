# COM-Sync · 多端同步 + 账号体系 (Multi-Device Sync & Account System)

- **状态**: canonical
- **日期**: 2026-06-14
- **产品线**: 商业线 (Commercial)
- **来源**: `40-gui-and-apps/35-gui-v2-vision.md` §2.1/§2.2；2026-06-14 路线图产品线重构

## 目标 (North Star)

让同一个用户在多台设备上无缝查看所有 Harness 项目的状态，无需手动 `git pull` + 重建投影。这是商业线的基石里程碑，为 COM-Mobile 和 COM-Team 提供账号和同步基础设施。

## 范围内 (In Scope)

### 账号体系

- 用户注册/登录（OAuth: GitHub/Google/邮箱密码）
- 设备绑定（一个账号 → 多台设备 → 多个项目）
- Electron 端登录流程（OAuth redirect + token 持久化）
- 权限：V2 阶段为**单用户**（自己看自己的）；多用户 → COM-Team

### 投影级同步

- **同步对象**: SQLite 投影 + 快照缓存 + 项目 registry + `harness.yaml` 配置
- **不同步**: 任务包原文档（Git 仓库本身不走此通道）、terminal scrollback/stdin/stdout
- **同步通道**: HTTPS + WebSocket（增量推送）
- **冲突解决**: 投影是派生数据，冲突时以 Git SoT 侧重建；配置冲突走 last-write-wins + 冲突标记
- **离线优先**: 离线时 V1 全部能力可用；联网后自动同步

### 隐私与加密

- 同步数据端到端加密（可选）
- 任务内容（可能含代码/secret）不经第三方服务器——只同步投影元数据
- `PublicPromotionGate` 规则同样适用于同步 payload

### GUI 集成

- V1 原型中的占位 UI 激活：
  - 侧边栏账号入口：从「本地模式」→ 账号菜单
  - 同步状态指示器：从固定「本地」→ 同步中/已同步/离线
  - 远程项目分区：从空态 → 来自其他设备的项目卡

### 付费网关

- 免费层：本地全功能 + 有限同步额度（如 2 台设备 / 3 个项目）
- Pro 层：无限设备/项目 + 优先支持
- 具体定价策略由商业 ADR 裁决

## 范围外 (Non-goal)

- 多用户/团队协作——COM-Team 负责
- 手机端——COM-Mobile 负责
- 文档级同步（同步 Git 仓库内容）——不做，Git 本身解决
- Relay 中转（跨 NAT live terminal）——需另立产品决策
- P2P 直连——技术储备，不在首版交付

## 开源/闭源边界

| 组件 | 开源/闭源 | 理由 |
|------|----------|------|
| Electron 登录 UI 组件 | 开源 | 本地工具组件 |
| 同步客户端 SDK（推/拉协议） | 开源 | 社区可自建同步服务器 |
| 同步服务器 | **闭源** | 商业基础设施 |
| 账号后端 | **闭源** | 运营成本回收 |
| 付费网关 | **闭源** | 商业逻辑 |

## 入口条件

1. GUI-V1 验收通过（Electron 客户端稳定可用）。
2. 商业模式 ADR 签署（付费策略、隐私政策、服务条款）。
3. 同步协议设计文档定稿（需新建 `harness/contracts/41-sync-protocol-contract.md` 或等价文档）。
4. `35-gui-v2-vision.md` §6 的 V2-1~V2-5 开放问题全部裁决。

## 验收标准

- [ ] 用户可通过 OAuth 注册/登录
- [ ] 两台设备（如 MacBook + Windows PC）安装 Electron 客户端后，同一账号下看到相同的项目列表和任务状态
- [ ] 设备 A 创建任务 → 设备 B 在 30 秒内看到新任务（在线状态）
- [ ] 拔网线后设备 A 继续正常操作；恢复网络后设备 B 自动同步到最新状态
- [ ] 同步 payload 不包含任务包原文档内容（仅投影元数据）
- [ ] 付费网关：免费层超出设备限制时提示升级，不阻断本地操作
